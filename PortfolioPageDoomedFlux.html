Overview

Created as participation for the Theana Game Jam, the theme was push and pull.

I wanted to experiment with UE5's physics and tried to make a horror game where the box the character would push and pull, would be needed for the whole level.

Responsibilities

Creating the game design (Game rules, mechanics, players, enemies, level progression).

Event planner, enemy placement, bomb/timer.

Level Design

Planar movement and first person vent crawling mechanics.

Managing the team.

Asset placement.

UI/UX. (UI animations)

All content developed via Blueprints

Japanese Translations

Synopsis/Demo

2.5D Horror/platformer where the theme is to push and pull the cube and use it to activate various panels across the space station. Be careful of the Infection!

Character Mechanics

Push/Pull: Player can push and pull a power box, it has multiple uses.

Kick: Player can kick the box which will kill enemies if they are hit.

Jump: Player can jump to reach different areas of the level

Fast Crawl (Vents Only): Player can hold a button to move quicker in the vents but will cost oxygen to perform.

Box/Level Mechanics

A Weapon: The box can be used as a weapon, if the player kicks it.

A Platform: The player can use the box as a platform to reach different areas of the level

A Power-Source: The box can be used as a power source which will turn on the lights, activate teleporters or elevators, and also it will become a bomb.

Enemy Mechanics

Zombies

Zombies will either crawl or walk towards the player at varying speeds.

Zombies will strike or bite the player if they get close.

New Skills Developed

Experimented with Unreal Engine 5's physics systems

UI Animations.

Using a game object to control lighting.

Adding controlled zones for 3D movement, and then the rest remained 2.5D or first person for vents.
